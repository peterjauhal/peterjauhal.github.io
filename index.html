First steps for my blog

  1. M-x org-static-blog-create-new-post
  2. You write there. It is saved in your "posts" directory of your blog.
  3. Once you are done, M-x org-static-blog-publish to rebuild the entire blog.
  4. Go to the directory where the generated files are. I believe we called it "assets".
  5. Do M-x magit-status to get into the Git interface.
  6. Type s in the Magit status to "stage" the unstaged changes. This means that you select which changes you wish to record in the history.
  7. Type c followed by c inside of Magit to "commit" the staged changes. This means that you now write an entry to the history log.
  8. At this point, Magit produces a Git commit message composition buffer. Describe in a few words what your change is, such as "Add new publication" or "Add journal entry for 2023-12-10".
  9. Once you have written the message, type C-c C-c in the message composition buffer to confirm/write the message.
  10. You are done recording the history. Now you are ready to "push" your changes, meaning to synchronise them with GitHub.

  11. Type P followed by p to push to GitHub (technically, you are pushing to the configured "remote").
